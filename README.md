# 🏠 Семейный бюджет: PET-проект с нуля

**Добро пожаловать в наш учебный проект!**

Мы — **Маша** ([@marianikana](https://github.com/marianikana)) и **Артём** ([@atema-hash](https://github.com/atema-hash)) — создаём с нуля систему для учёта семейного бюджета.

## 🎯 Цель проекта

Главная цель — **не просто получить программу, а самостоятельно пройти ВЕСЬ путь разработки** от идеи до работающего приложения, которым мы будем пользоваться дома.

Мы учимся на практике:

1. **Анализировать:** формулировать задачу, выявлять требования.
2. **Проектировать:** описывать сценарии, рисовать диаграммы, проектировать базу данных и интерфейсы.
3. **Разрабатывать:** писать код как на фронтенде, так и на бэкенде.
4. **Эксплуатировать:** разворачивать систему на домашнем компьютере, обслуживать и обновлять её.

## 📈 Наш план (технологический стек будет уточняться)

### 1. Анализ и проектирование:
- Постановка задачи и формализация требований.
- Описание сценариев использования (User Stories) и sequence-диаграммы (PlantUML).
- Определение сущностей, атрибутов и связей.
- Проектирование структуры реляционной базы данных (ER-диаграмма, DBeaver).
- Создание эскизов интерфейса. (Figma, Draw.io)
- Выбор инструментов, технологий и фреймворков на основе требований к компонентам.

### 2. Разработка:
- Верстка HTML/CSS страниц по скетчам.
- Разработка backend-логики (Java).
- Разработка frontend-логики (JavaScript).
- Настройка базы данных (PostgreSQL).

### 3. Внедрение и сопровождение:
- Развёртывание системы на домашнем сервере (on-premise).
- Тестирование и сбор feedback от семьи.
- Поддержка, исправление ошибок и добавление нового функционала.

## 🗂 Структура репозитория

Структура будет наполняться по мере работы. Предварительно планируются следующие разделы:

- `/proto` — прототипы: пробы, верстка, примеры, ошибки.
- `/docs` — документация: требования, диаграммы, скетчи.
- `/backend` — исходный код серверной части.
- `/frontend` — исходный код клиентской части.
- `/database` — SQL-скрипты создания и миграции БД.
- `/deploy` — конфигурации и скрипты для развёртывания.

## 👨‍👩‍👧‍👦 Для кого этот проект?

В первую очередь — для нас, **Маши и Артёма**, как полноценная учебная практика. Во вторую — для нашей семьи, как полезный инструмент для ведения бюджета. Мы открыты к конструктивным советам, но на данном этапе это наша личная учебная лаборатория.

## 💡 Философия проекта

> **«Понять, как это работает, можно только собрав это самому».**

Мы не ищем готовых решений и шаблонов. Каждая строчка кода, каждая диаграмма и каждое решение — результат наших проб, ошибок, обсуждений и совместных решений.
